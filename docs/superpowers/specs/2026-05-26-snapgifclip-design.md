# SnapGIFClip 設計文件

**日期**：2026-05-26  
**工具名稱**：SnapGIFClip  
**類型**：Windows 工具（Python + tkinter）

---

## 一、工具概述

按下快捷鍵後，框選螢幕任意區域，設定秒數（或手動停止），錄製並輸出成 GIF 和／或 MP4。附帶影像編輯功能，支援載入任意 GIF/MP4 做裁切、調速後匯出。

---

## 二、技術棧

| 用途 | 工具 | 說明 |
|------|------|------|
| 截圖擷幀 | `mss` | 比 PIL.ImageGrab 快 5 倍 |
| GIF 編碼 | `Pillow` + `imageio` | 品質優於純 Pillow |
| MP4 編碼 | `ffmpeg`（本地） | `bin\ffmpeg.exe`，launcher 首次自動下載 |
| 熱鍵監聽 | `pynput` | 僅在 App 執行中有效（非系統全域） |
| UI | `tkinter` + `ttk` + `sv-ttk` | sv-ttk 未安裝自動 fallback 系統主題 |

## 四、模組職責

| 檔案 | 職責 |
|------|------|
| `main.py` | tkinter 主視窗；建立三個 Tab；`pynput` hotkey 監聽；背景執行緒與 UI 之間的 queue 通訊 |
| `recorder.py` | 接收錄製參數（座標、fps、秒數、格式）；mss 截圖迴圈在背景執行緒；完成後呼叫 Pillow/imageio 產 GIF、ffmpeg subprocess 產 MP4 |
| `overlay.py` | `FullscreenOverlay`：全螢幕半透明暗幕，拖拉選框，回傳選取座標；`RecordingBorder`：貼合選取區域的紅框 Toplevel，錄製期間常駐 |
| `editor.py` | 影像編輯 Tab 的所有邏輯：載入 GIF/MP4、解幀快取、Canvas 預覽播放、裁切/調速計算、估算容量、ffmpeg/Pillow 匯出 |
| `config.py` | 讀寫 `src/config.json`；任何設定變更即時寫入（自動儲存） |

## 五、錄製流程

```
1. 使用者按下快捷鍵（pynput 監聽）
2. overlay.py → FullscreenOverlay 出現（全螢幕半透明暗幕）
3. 使用者拖拉滑鼠選取區域 → 放開後顯示選框確認框線
4. （若設定開啟倒數）overlay 顯示 3-2-1 倒數
5. overlay.py → RecordingBorder 出現（紅框貼合選取區域）
6. recorder.py 背景執行緒開始：mss 按設定 fps 截圖
7. 主 tab 顯示進度條（已錄 X.X / N 秒）+ 「提早停止」按鈕
8. 達到預設秒數 or 使用者再次按下快捷鍵 → 停止截圖
9. RecordingBorder 銷毀
10. 主 tab 切換為「處理中... 正在產生 GIF/MP4」
11. 編碼完成 → 主 tab 顯示「最後輸出」區（檔名 + 開啟資料夾）
```

```
┌─ 載入檔案 ───────────────────────────────┐
│  [ C:\...\output.gif            ] [ 開啟 ] │
│  GIF｜640×360｜10.0 秒｜15 fps            │
└──────────────────────────────────────────┘

┌─ 預覽 ───────────────────────────────────┐
│  ┌──────────────────────────────────┐    │
│  │                                  │    │
│  │         影像顯示區（Canvas）       │    │
│  │         縮放至符合寬度             │    │
│  │                                  │    │
│  └──────────────────────────────────┘    │
│    [ ▶ 播放選取範圍 ]  1.5 / 6.0 秒      │
└──────────────────────────────────────────┘

┌─ 裁切範圍 ───────────────────────────────┐
│  起始 ━━●━━━━━━━━━━━━━  2.0 秒           │
│  結束 ━━━━━━━━━━●━━━━━  8.0 秒           │
│  → 選取時長：6.0 秒                       │
└──────────────────────────────────────────┘

┌─ 播放速度 ───────────────────────────────┐
│  ○ 0.25x  ○ 0.5x  ● 1x  ○ 1.5x  ○ 2x  │
│  → 實際輸出時長：6.0 秒                   │
└──────────────────────────────────────────┘

┌─ 匯出設定 ───────────────────────────────┐
│  輸出格式：  ○ GIF   ○ MP4   ● 同來源格式 │
│  預估容量：  約 3.2 MB（估算值）           │
└──────────────────────────────────────────┘

                  [ ✂ 匯出 ]

┌─ 輸出結果（匯出後顯示）──────────────────┐
│  ✅ 完成：output_edited.gif               │
│  [ 開啟資料夾 ]                           │
└──────────────────────────────────────────┘
```

**影像編輯邏輯**：
- 拖動起始/結束滑桿時，預覽自動跳到該時間點的 frame
- 播放按鈕：循環播放選取範圍，速度依播放速度設定
- MP4 載入時背景解幀（顯示「載入中...」），解幀縮小為預覽尺寸（不影響匯出品質）
- 容量估算：GIF 取 5 個 sample frames 快速試編後外推；MP4 依 CRF → bitrate × 時長計算；滑桿移動後 0.5 秒 debounce 重算
- 匯出檔名：原始檔名加 `_edited` 後綴，存到設定的輸出資料夾

## 八、ffmpeg 自動安裝流程（launcher.ps1）

1. 檢查 `bin\ffmpeg.exe` 是否存在
2. 若不存在 → 顯示說明：「即將下載 ffmpeg，用於 MP4 輸出，約 80 MB，看不懂可以把這段說明丟給 AI 查詢」
3. 從 GitHub Releases 下載 Windows 靜態版 ffmpeg zip
4. 解壓 `ffmpeg.exe` 到 `bin\`
5. 後續所有 subprocess 呼叫使用 `bin\ffmpeg.exe` 絕對路徑，不修改系統 PATH

## 十、已知限制與注意事項

- 快捷鍵僅在程式視窗開啟時有效（非系統全域 hotkey）
- GIF 最大色彩數 256，超過此限為 GIF 格式本身限制
- 影像編輯 Tab 載入大型 MP4 時，解幀需要數秒；預覽畫質會縮小，匯出品質不受影響
- `mss` 在多螢幕環境下，選取座標以主螢幕座標系為準
